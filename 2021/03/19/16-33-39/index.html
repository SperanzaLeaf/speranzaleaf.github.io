因为原来那个博客园博客塞满了OI时期的题解，笔记和各种记录，多得我也懒得收拾了(咕咕咕

·从零开始的博客搭建过程及其中可能的问题

hexo+Github repo

要装hexo需要npm，要装npm需要nodejs，递 归 安 装。当然直接去nodejs官网是会白给的，请使用清华镜像，下下来以后可以加进path里方便以后用。npm装好之后装hexo

npm install -g hexo

于是gg了

因为众所周知的原因npm的官方源下载速度过于可观，需要先换成更慢的国内镜像

npm config set registry https://registry.npm.taobao.org

之后hexo init初始化hexo，当然正常来说这步也会gg，需要自己动手进行这几个步骤，模拟hexo init：
1.去github把hexo starter的zip文件下下来并解压到一个文件夹里
2.在这个文件夹中运行
npm install
3.补两个module(我记得是这俩，可能有问题，请根据实际情况补
npm install nunjucks
npm install hexo-util

之后hexo s,去localhost看看，应该已经可以了

最后传到github repo，去配置文件_congif.yml里找到deploy字段写上自己的仓库地址，注意git现在默认的分支是main(背后的故事令人暖心，草)

最后

hexo clean
hexo deploy

完工

可能的问题：更新时博客时spawn failed
解决办法：把git加到path里
发现好像tm就是常见网络问题
